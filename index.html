DataBinding之『简单使用』

什么是Data Binding

DataBinding可以说是一种Android平台MVVM的框架，直接将数据和视图进行绑定。

Google很早就推出了Data Binding Library, 不过一直是Beta版，存在着一些Bug。 这次Google IO 大会（2016年）上发布最新Data Binding Library，现在DataBinding已经是Production了，你可以放心的玩耍了！！

使用Data Binding Library的环境要求

  • 它是Support Library，Android 2.1(ApI Level 7+) 都可以使用

  • Android Gradle Plugin 1.5.0-alpha1以上版本

  • Android Studio 1.3以上版本

简单使用步骤

Step0

新建一个Android Project，修改app module中的build.gradle文件，加入以下代码：

android {
    ....
    dataBinding {
        enabled = true
    }
}

注意： 如果你的工程依赖了一个使用了Data Binding 的Library，你的app module中build.gradle依然要加入以上代码

Step1

准备你的ViewModel类，有两种写法，如下样例：

第一种写法

/**
 * Created on by yydroid 8/17/16.
 */
public class User {
    private final String firstName;
    private final String lastName;
    public User(String firstName, String lastName) {
        this.firstName = firstName;
        this.lastName = lastName;
    }
    public String getFirstName() {
        return this.firstName;
    }
    public String getLastName() {
        return this.lastName;
    }
}

第二种写法

/**
 * Created on by yydroid 8/17/16.
 */
public class User {
    public final String firstName;
    public final String lastName;
    public User(String firstName, String lastName) {
        this.firstName = firstName;
        this.lastName = lastName;
    }
}

读《小狗钱钱》所想

思考

  • 讲道理有很多种方式，但是我想，最容易被人接受和记忆的，恐怕非『故事形式』莫属了。对于《小狗钱钱》这本书，你可以说它是一本理财类的书籍，但是它更是一本童话与现实结合非常紧密的一本书

  • 我想，对于第一次接触『理财』这个话题的人来说，可能你觉得『理财』觉得很高深，其实，它也是非常容易入门，它的入门也是有一套基本的套路，例如：明白钱的意义、设定目标、储蓄、控制支出、增加收入、尽量少借贷（非良性）等等。

  • 对于理财的高深篇，我觉得是投资篇，『投资』是理财二字中『理』的一种更高深的方式，投资本身就是一门很高深的学问，需要花更多的时间、精力去研究，它是一种情商和智商相结合的学问，很多人称之为『财商』

  • 对于『理财』『投资』，你不能轻视他，但是你又不能太觉得它们太高深莫测，我个人觉得，你得以平和的心态去看待他们，如同『成熟的心智』处理突发事件一般，如沐春风。但是你得认真，你需要用『仪式化』对待理财，比如：你可以真得准备一本梦想相册，把你想实现的愿望的『画像』放进去。不管是自己画，还是去寻找一些相片，把它们放进去。让自己每天看到。这样你才有动力。比如，我想要买一栋大房子，但是，这不够具体，你需要具象化你的房子，有几扇窗户，几扇门，几间房，装修是什么样子的，墙面是什么颜色，把它们统统具象化。

  • 投资理财是一条漫长的路，但是你得有一个目标，给自己多少时间，拥有多少资产。一般以3-5年为一个期限。总之，这个期限的时间不宜过长，也不宜多短。赚钱的速度肯定越快越好，不然你超越不了金钱的逃逸速度（此词来源于李笑来）。如果这个期限太短了的话，很容易让人浮躁。

  • 打开自己的心扉，让自己多接触一些好的理念，别太封闭。多多请教你觉得『富有』的那些人，对『富有』的定义，可能每个人都不太一样，但是没有关系，你自己把控，相信自己的眼光。因为这种看人的眼光，在未来发展的道路，你肯定需要。

  • 多多学习一些『经济』『法律』等等方面的知识，这些可能不是你以前的专业，但是没有关系。为了成功，为了你的财富目标，这些你必须去学。学这些不为别的，是为了不被其他人误导，是为了增加自己解决问题时搜索的知识范围

  • 在投资任何一种『投资品』时，你先得深度了解它，不然就是『赌博』。比如，你可能会去投资股票，那你得问自己，股票是什么？股票是如何运作的？收益是如何产生的？你是否适合股票投资？股票投资有套路吗？

Mac说之『打造自己的命令终端』

Mac的GUI和命令行的完美结合让广大程序员们对它情有独钟。也正是因为这样，命令行的重要性可想而知。本文的重点就是告诉你如何打造自己的命令行。

第一步：下载iTerm

iTerm 这个工具我在Mac说之『我的工具集』中提到过，没安装的，赶紧下载安装吧，官网地址：https://www.iterm2.com/

第二步：切换成zsh

打开iTerm, 输入：chsh -s /bin/zsh

第三步：安装oh-my-zsh

  • 安装git (Mac系统自带，可以跳过)
  • git clone git://github.com/robbyrussell/oh-my-zsh.git ~/.oh-my-zsh
  • cp ~/.oh-my-zsh/templates/zshrc.zsh-template ~/.zshrc

第四步：配置zsh

完成以上三步，你的终端基本上配置完成了。

如果还需要按照自己的喜好进行配置，你只需要找到当前系统用户所在的目录，找到.zshrc文件，按照一定的要求配置这个文件，你就能打造自己的个性终端了。

浏览这个网页https://github.com/robbyrussell/oh-my-zsh/wiki/themes, 你能看到所有不同主题的预览。

两个神一样的编辑器

有人说Emacs像神的一样的编辑器，也有人说VIM像神一样的编辑器，我说：它们都是有『思想』的编辑器

Emacs

它采用组合键的方式，它的使用直接反应着『函数式编程』的思想。它的『真身』是一个编辑器，但是你可以将它打造成一个IDE。并且，你可以用它来收发邮件、浏览网页等

Emacs官网：https://www.gnu.org/software/emacs/

Vim

它采用三种模式：命令行模式、编辑模式、视图模式，结合这三种模式，你的手可以不用去碰鼠标，全过程用键盘操作。你可以将它打造成一个IDE

Vim官网：http://www.vim.org/

掌控时间最好的方式是构建自己的维度

在当前这个爆炸的信息时代，很多人都从中受益了。但是同时，信息的繁多带给人们的选择也会太多，需要不停地筛选。如果每天都这样，很容易让人陷入一种混乱的状态：这一天太忙了，但是一细想，不知道忙了什么。

为此，你可能会觉得你没有将时间管理好，你需要学点时间管理方面的知识，你搜索网络，你会发现有太多时间管理方面的文章、书籍，此时你又陷入了选择中……

当然，我也被这个问题困扰了很久，每天晚上回到家，身心疲惫，但是细问自己：今天忙了什么呢？我是一脸困惑。

为此，我痛定思痛，花了一段集中的时间去寻找『到底怎么样才能减少我们的这种混乱』的方法。 我看了很多跟时间管理相关的书，例如：《超级时间整理术》、《时间，快与慢》、《时间的朋友》、《番茄工作法》
当然同时也顺带看了一些与『自我管理』这个主题相关的其他一些书籍，看了很多微信公众号的文章，听了很多的录音，想了很多，给自己狂轰乱炸的这段时间，我慢慢看到了自己的『时间系统』成型了。

其实，混乱的根本原因，你的生活没有章法，没有章法的原因，是因为你没有『仪式化』你的生活，而『仪式化』生活的简单方法就是：将你的生活划分维度，这里我以我的『生活维度』为例。

现在，我将我的生活基本上划分以下几个维度：

  • 时间

  • 理财

  • 运动

  • 家庭

  • 创业

  • 技术

当划分好以上几个维度后，我开始细化如何掌控这些我必须花费的时间：

时间

对于时间，我觉得我挺同意『李笑来』老师的观点：时间是不可管理的。我想，理解李笑来老师的这句话，你不能只停留在字面上，你得探究出这句话背后的含义。我的理解是：时间不能用管理一词来描述，你得用『掌控』。你必须掌控时间，如果，你掌控不了时间，那你就只能让时间掌控你。举个例子，当你和朋友聊天的时候，你可能会被一些暂时不重要的微信信息吸引过去，此时，我可以说：时间掌控了你，为什么呢？要么这次谈话不重要，你正在浪费时间；或者这次谈话很重要，但是你却被不重要的一些信息打断了，导致这次谈话的效率大大折扣；不管从哪个方面你都被时间掌控着。

理财

俗话说，你不理财，财不理你。金钱是我们在城市里面赖以生活的基础，对于我们的生活，它是处于渗透状态，如果这么重要的东西你都不理，那还理什么。而且，金钱是可以换时间的，怎么理解呢？举个例子：此时你正在家里处理一个非常重要的文件，但是吃午饭的时间到了，此时你可以叫上一个外卖，只需要花上一丁点的外卖配送费；再举个例子，你平时家里的东西坏了，你可能不会亲自上阵去修理吧，你可以请一些专业维修人员来帮忙修理，此时，你只需要付一点维修费。

都知道时间很宝贵，时间如生命，能够换来时间的金钱，你怎么能不好好对待呢。

运动

由于我的职业是经常坐着办公，避免不了，我的颈椎和背部每天会酸痛胀痛。所以我必须花时间去运动。而对于生命，不就是『生命在于运动』

当然，运动是和健康挂钩，投资自己的另一个方面不就是投资自己的健康嘛

家庭

家庭是事业的基石，一个和美的家庭，会让你的工作生活格外美好。这个我相信你肯定认同。为此，我非常清楚地知道，我将必须花时间在家庭上。我在处理家庭关系时，把家庭当做一个别样的『团队』对待，如同创办企业一样，你需要不停地给家庭输入正确的价值观，描绘以后的发展方向，让家庭的每一个成员为之努力。其实，家庭关系是可以简单处理的。别总被古话所迷惑，忘记那『清官难断家务事』吧。

创业

以创业的心态对待每天你的工作。以员工的身份对待工作，那是为别人打工；以创业者的心态对待工作，那给自己打工。别觉得这句话是为了麻木自己，其实，你的努力，你自己知道，怎么可能只是给你现在的公司带来好处，而对你一无所获呢。

每天多看一些创业的文章，创业公司的发展动向，创业相关的书籍以及创业相关的人，这些对你是有好处的。

技术

我是一名技术人，技术是我的兴趣，也是我当前的专业，对待它，我得花时间。

其他的一些小技巧

  • 给自己一点时间，什么都不想，什么都不做，没有目的。

  • 发现自己又开始混乱了，立即停下手头上所有的事情，去找个静静的地方走走

  • 当做事的时候，关掉所有的通信设备和通信软件，此时的时间应该是你的。可能你会被人打扰，但你得学会拒绝。你可以说：我回头处理，我回来找你，可以吧。

  • 一定要给你的自己时间进行切片，切片可以不均匀，但是一定要做。因为，你会疲劳的，你需要休息

Mac说之『你真得该把你的电脑换成Mac』

在之前的一篇文章《我认为的程序员的自我修养》中，我提到修养之一：使用Mac或者其他的类Unix系统。

此时，我不想再啰嗦地说：『其实，考虑到价钱，你可以买个其他的笔记本，然后装上Linux，例如：Ubuntu、Fedora等等一些其他的类Unix操作系统』。好吧，我们还是直接点，换Mac吧，别犹豫了，你会觉得很值得的。

为什么要换Mac，我得说几点，对于这几点，也许只有当你用上Mac之后，才能感觉出来。当然有些观点，是不用非得等到用上Mac之后才需要很好地理解的。

工具即效率

人类和动物的区别有很多，其中一个最普遍被知道的观点是：人类与动物的区别在于工具的使用。

Mac是一款工具，一款被程序员们（高级的思考者）公认的好工具，它是能极大程度提高效率的工具。

价格真不贵

其实，这个观点涉及到投资的话题，我想你总是听说过一些观点，例如：投资自己是最好的投资『这个观点我是非常认同』；股票有风险，入市需谨慎『这个要用不同的角度看待，对于一些高手，一些会玩股票的人，风险这个东西，他们是可以驾驭了，而对于不会玩的，对于风险的态度可能是：预防』等等。我们这里不讨论上述这些关于投资过于深入的话题，学习理财，学习投资不是一天两天就能学会，这里我想说的是：对于好工具的投资，肯定没有错。

经常听朋友说，Mac太贵了，要一万多RMB呢，这句话对吗？ 我想这是靠感觉去衡量的，而不是靠理性去衡量。

当人们说出这句话的时候，我想肯定没有认真分析过，暂时花的这一万RMB将会带来多少生产力的提高，将给你节省多少时间，将给你带来多少无形的财富。

人们总说时间就是金钱，我想时间的价值肯定大于金钱，因为钱可以去赚，但是时间过去了就真过去了。而时间这么宝贵，只考虑这一点，Mac的价格还贵吗？

你可能愿意花大几千块买个电视机，也可能花个大几千买个自行车等等，但是为什么觉得花个大几千买个Mac，你就觉得贵了呢？

乔布斯团队设计出来的东西，我就得用？凭什么，我就不用。可能，有些人会有这些想法，而我的观点很简单：一个人被世界公认为是神一样的人物，我想肯定是因为他做出什么成就，做出了伟大的产品。 而Mac正是他们团队搞出来的电脑

重拾《幸福了吗？》

下午闲来无事，静坐在书架前，随手翻翻书架上的书，从技术到人文，可能这也是我当前最大的乐趣之一了。

随手拿起一本书，扫过几页，又重新放下；又拿起另一本，再用眼睛扫过几页，再放下，好像没有目的，没有意图的这个时候，感悟反而是最高的，因为看过的文字，脑子基本都记住了。

今天拿起了四年多以前买的一本书《幸福了吗？》，这本书的作者是白岩松。随手翻到了最后一章，竟然津津有味地看了起来，从后往前看，一口气看到了第十一章《美国，美吗？》。还记得，四年前，刚在京东上买回这本书的时候，白岩松的文字，无论我怎么用力看，都看不进去，基本上是看了一会儿，就会走神。走神的原因是没有感觉，心灵并没有得到最大程度的碰撞。现在想想，可能那时候，还太年轻，太轻狂无知。依稀记得这本书的内容当时并没有吸引住我，但是这本书的名字倒深深地烙在我的脑海里，虽然那时候，我也并没有问自己：我幸福吗？但总觉得这本书其实有内容，只是这本书还不适合我当时的心境。

所以，这些年，我看书都有一个习惯：不能很快吸引我，读起来不是很有感觉的书，我会立马放下，哪怕这本书是多么的经典，多么地被朋友推荐。我想，读书本质上是一场对话，是穿越时空与作者进行对话，如果，作者说的，你看不懂，那可能你们不在一条道上，或者说，你们不在一个境界中，如果只在当下一味地摄取作者的文字，反而效果不是很好。还不如等你的想法，你的觉悟到达了和作者基本的水平线上，或离水平线不远处，再去进行对话，效果会更好。当然，有时候，你不去看某一本书，某一类书的原因，可能你自我感觉你不需要，但是你的感觉未必是对的，所以，为了避免这个问题，我基本上没有给定自己看书的界限，只要是好书，只要是经典，我一定都会翻它几页，然后再去判断当下是否要去读它，读之，会随身携带，此时不读，会放到书架上。

再说说，《幸福了吗？》后面几个章节中得到的一些东西，和大家分享：

  • 白岩松用它的视野书写了一代人的成长与困惑，这里面酸甜苦辣，读之深有味道。

  • 相信大家都很熟悉的“道可道，非常道”，这是习惯的断句方式，但也可以用另外一种断句方式：“道可，道非，常道”。虽不知道哪一种更好，我想，句式的错与对，已经无所谓了，只要你明白它的含义

  • 随着作者文字的流淌，一张张过去的图片展现在眼前，这让我看到一个人的脸庞从“稚嫩”变得“富裕”，多出来的除了肉，剩下的都是内容。我不禁想到，10年后，我的脸上，会展现出一种怎么样的感觉。

  • 之前，在我看来，白岩松是个名人，是央视的一名著名的主持人，但是在他的书里，我觉得他离我很近，他也曾年少轻狂过，也傻过，与我很近很近

我认为的程序员的自我修养

看过星爷《喜剧之王》的朋友，我想你们肯定记得星爷捧在手里的那本书《演员的自我修养》。说实话，我很喜欢星爷的电影，而且星爷的电影百看不厌，每看一遍，总有些不一样的感受，经典就是经典。《喜剧之王》这部剧我也看了很多遍，没记错的话，应该不少于五遍。前段时间，为了放松，又重温了一遍。当看到星爷每天都看那本放在身边的《演员的自我修养》的时候，心里不免生出些许感受，我想到，作为一名程序员，也需要有自己的修养。其实，有时候，修养都不用多么高大上，而都是在小处。

修养之一：热爱自己的每一行代码

平时的工作生活中，总听到一些IT行业内的人士说道：刚开始写写代码，后面尽可能去转管理，总会有一堆的理由，在此我就不多说了，反正这些理由对我而言，都是千篇一律。曾经我也一度觉得挺合理，但现在想想，觉得可能需要用另外一种角度去看。我就在想：为什么不能够干一辈子程序呢？如果你喜欢你的代码，你是一名非常专业的资深的程序员，你是这个领域的专家，难道，你后面的路就会有瓶颈？你想呀，人家寿司之神小野二郎一辈子只做寿司，过得不也挺好，而且享誉世界。人家热爱的是寿司，因为热爱，所以持续去做。所以，别怕，同学，尽一切去爱你的每一行代码，爱你现在所从事的行业，你想呀，能用自己的爱好去谋生，是多么的光荣。朋友，当别人问你做什么的时候，你应该能够自信并且自豪地说出：我是一名程序员。

修养之二：购买正版软件

首先，我得承认，我用过盗版软件，曾经我为了破解软件而去寻找很多方法，当然，在天朝，这种事情司空见惯，现在，我已经改过自新了，并且为此也内疚了一段时间。

大概就在去年，我慢慢的意识到：我是靠开发软件谋生，而那些软件开发同行们辛辛苦苦开发了这么一款好用的软件，能极大地帮助我们提高效率，如果，作为同行的我不买单，那真是太可耻了。

修养之三：使用Google

经常对朋友说：用google吧，别在用百度了。但朋友经常会说：使用google需要翻墙，太麻烦了；百度也能搜到呀；有些八卦新闻google还真搜索不到等等。
为此我的想法很简单，如下：

  • 翻墙太麻烦？对于一个开发者，翻墙这个破事都麻烦，那还搞个什么程序。
  • 说百度也能搜到，这个理由，我很还真没法反驳，确实百度也能搜到一些资料，但是你好好比较下google和百度搜出来的结果，你就会发现，二者的搜索结果差异性太大了，前者永远是权威性的，后者永远都是八卦性的。
  • 八卦新闻google有时候搜索不到。这个呢，不是说google搜索不到，只是google将那些八卦性的东西放到后面几页，前面几页都是一些权威性的介绍，而不是八卦。但是，你再细想下，用google不是更好嘛，八卦搜不到，那平时就不用分心了，还能让我专心去学习。所以，对于咱们搞“学术”相关的，还是多用用google
  • ……

修养之四：去花钱买个VPN

上面说到，翻墙太麻烦，那是不是因为你平时找各种翻墙大法，今天用这个，明天用那个，结果时间都花在找VPN上了。为什么不买个正规的VPN呢，像现在我用的云梯还是蛮稳定的，一年花个几百块钱，不知道省了我多少时间。都说时间宝贵，几百块能节约这么多时间，岂不是很值得。

修养之五：使用Mac或者其他的类Unix系统

拥有一个开放的心态挺难，而做到时刻保持开放，更是难上加难。为此，大概在3年前，我已经基本上不在Windows平台上开发了，我的想法很简单：做一个开放的人，先把自己手头上的工具换成开放的。当然，不是说，windows不好，也不是说在Windows上开发程序就不行，更不是说Windows程序员不牛逼（Windows平台有很多牛人，不用说别的，微软里面有着一堆的高手），只是，对于学习编程的我们，Windows隐藏了很多细节，以至于我们看不见摸不着。而Linux就不一样，所有的东西你都能看见，从内核到上层GUI，你都可以拆来拆去。在平时的开发中，从环境配置到程序开发，都需要你自己动手。

修养之六：时刻保持开放的心态

这点体现在平时和同事、朋友的沟通上，咱们做程序的，经常会犯一个毛病：认为自己的思路就是对的，可能也是最好的，别人的思路呢，不能说错，反正就是经常听不进去。我想，其实，开放的心态在任何事情上，都是需要的。而保持开放，做一个善听者，需要时刻地提醒自己。我平时的做法是：先听听，再想想，多问问，再看看，然后再去说说。

总结

我认为的修养其实用几句话就能讲清楚：一个热爱程序的程序员，使用着Mac，用着一些正版的提高效率的软件，开发着自己程序，当遇到一
些难点的时候，使用着google查阅着各种资料，并以一种开放的心态和同事、朋友沟通，阅读着开源的代码
