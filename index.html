给JavaScript初学者的鸡汤

By YYDroid

Published 2016-03-09

纵观现阶段的Web前端开发，诸多的框架，诸多的开发工具，诸多的技术点，如此繁杂，让人应接不暇。随着时间的推移，各种新技术又层出不穷，特别是近两年来，Hybrid这个看似一种”新平台“的兴起，又催生出了一波前端技术潮。我想，很多时候，这些肯定会让人感到恐惧不安，感到疲惫；也会让一个刚进入Web前端开发阵营的开发者感觉无可适从。但我觉得，不要被这些吓到，也不用过度理会这些，这些只不过是由心法衍生出来的招式。要把自己的关注点移到心法上，那心法又是什么？

心法

如果去问开发者，计算机的心法是什么？ 无论他使用哪一种语言，无论从事哪一种平台的开发，我相信他说出来的名词可能是：算法、数据结构，又或者是底层原理、操作系统原理、数据库原理等等；那Web前端开发中的心法又是什么呢？首先，它依然拥有上面的那些名词，但是它也有特定的上层心法：HTML、CSS、JavaScript,当然还有那高深的Webkit。在心法中，JavaScript又是怎么样的角色呢？而它在我心里，是一个利器；它也是接下来所有篇章的重点。

JavaScript

以上两段内容的铺垫只是为了引出JavaScript这个杀手锏；如同武侠小说，当武林高手出场的时候，总得弄出点什么来衬托，比如：美女。

JavaScript是什么

如同C语言、Java语言，它也是一种计算机编程语言；但是它和C、Java不同，它是一种解释型的脚本语言

JavaScript运行环境

一句话就能概括它的运行环境：拥有JavaScript解释器的平台。我敢保证：你的个人PC上肯定具有JavaScript的运行环境，只要你装过浏览器。现阶段，JavaScript的运行环境有这些：各个浏览器（IE、Firefox、Chrome、Safari、QQ浏览器等等）、node.js等等

JavaScript使用范围

  1. 所有的动态网站（我想，现有很少能看到纯静态的网站。夸大点说，你每天能看到的网站，99%的概率使用了JavaScript）
  2. 服务器端（node.js搭建的server）

JavaScript的社区

可以这么说，网上关于JavaScript的资料遍地都是，只要你想学，只要你去找，很容易就找出一堆资料。废话不多说，看github上关于JavaScript的资料：https://github.com/search?utf8=%E2%9C%93&q=javascript

JavaScript的框架

  1. jQuery
  2. Zepto
  3. vue
  4. Angela.js
  5. ……

JavaScript框架如此之多，妈妈再也不用担心你学JavaScript了，对不对。呵呵

总结

从上面的内容，我想你应该能得出以下结论：

  1. 精通Web前端开发，必须精通JavaScript
  2. JavaScript语言的调试环境非常方便
  3. 不愁找不到JavaScript学习资料
  4. 遇到问题，不愁找不到大牛帮忙解答
  5. 想提高生产率，不愁找不到一个合适的JavaScript框架

学习资料推荐

一次淘宝购物引发出对产品的思考

By YYDroid

Published 2015-11-01

背景

不久前，在淘宝上买了一个书架，自己动手组装了一下，我说说自己的一点感受：我作为一个小白用户，可能不是太小白，因为是有组装经验的;但当我拿到此款产品的时候，完全没有头绪，也完全不知道从何开始，感觉一团乱。

思考

淘宝上卖书架，卖同类商品的那么多，如何在繁多的商家中脱颖而出，必须找到和其他商家的差异化，找到自己的核心竞争力。那差异化在哪儿呢？我想到几点：

  • 一分钱一分货，人们都知道这个道理，人们也都有想着能以最少的钱拿到最优质的产品和服务；所以对于我们商家，必须是最小成本保证商品的质量和细节（木材的优质可能无法提高，因为成本放在这；但是可以尽量提高烤漆的质量，提高烤漆的等等细节）

  • 淘宝上卖商品，还有个差异化的地方就是：服务，任何地方都一样；海底捞能这么火，这么成功，卖的就是火锅，但是为什么这么成功——差异化的服务。那对于我们商家，差异化的服务又在哪儿呢？听我如下分析：

诉我不才，可能动手能力不是太强；作为一名互联网产品开发，我先说说此款书架的问题：

  • 我买的ABCD四个规格的柜子，店家寄回来的就是一摊木材，我拿到木材毫无头绪，因为不知道哪一块木头对应哪一个规格的柜子，为此，我觉得可以分四个包裹，ABCD分别对应一个包裹，这样对小白用户绝对有利

  • 当根据ABCD分成四个包裹后，大类区分完成了，但是还有一个问题：不知道哪一个木板对应哪一个区域，所以，最好给每一个木板做一个标记（可以简单的标记为1、2、3…）

  • 当有了标记后，得出一份稍微详细的组装说明书，这个说明书不能只是简单地说明组装步骤，有几个重要的方面需要说明：

    • 几个小部件的重要性，必须在说明书突出，因为一些小白可能不知道它的重要性以及组装顺序的合理性
    • 告诉用户，哪些关键点必须注意，否则会影响安装质量，有点东西安装错了，会导致木板损坏，很难恢复（例如：塑料螺帽安装到木板的小孔中的时候，不能砸得太狠，必须保证跟木板齐平，突出来会让木板衔接不密封，太深了会让螺丝无法卡在旋转螺帽中）

总结

不亲身体验自己的产品，不身临其境地体验自己的产品，你根本不知道自己的产品的问题。知道用户在想什么，用户需要什么，才是“用户至上”的意义

编程语言分类

七牛云存储上部署iOS应用

By YYDroid

Published 2014-07-29

可能很多人都用过Dropbox，但是很不幸，它最近被有关部门封了。其实，我很想知道封它的理由，但是也不关心，只是有点愤慨。所以，找到国内的七牛。在此也非常感谢七牛，它真得很帮开发者们很多忙；感谢的话嘛，不能说多，转入正题，如何利用七牛部署iOS应用：

Step1

稍微观摩一下七牛的网站http://www.qiniu.com，看看它是什么，长啥样，在IT界大概是属于哪一个领域的公司，and so on！！！我相信很多人正在用360网盘、百度网盘、金山网盘，这个七牛貌似也做网盘，有什么不同呢？当然，我只是提出了一个问题引子，你可以给自己再多提几个问题

Step2

注册一个账号。很荣幸，七牛提供了注册免费送空间的服务，新手上路步骤如下：https://portal.qiniu.com/tutorial/index

Step3

我相信经过第三步，你差不多已经会用七牛了。接下来，咱们可以干正活了。

  1. 上传你的ipa程序安装文件，两张程序的icon图片（分辨率分别为：57x57，512x512）

  2. 撰写你的plist文件：我想对于iOS开发者，这个文件不用解释，但是对于非iOS开发者，简单点说，它就是个xml文件，它的配置是固定的，按到苹果的规范走，iOS会自动帮我解析这个文件，我贴下我的代码（这个plist文件在下载中的具体作用，在下面的第3小步中你会看到的）：

    <plist version="1.0">
<dict>
 <key>items </key>
 <array>
   <dict>
     <key>assets </key>
     <array>
       <dict>
         <key>kind </key>
         <string>software-package </string>
         <key>url </key>
         <string>http://corptravel.qiniudn.com/ipa/Corp_FarEastHorizon_3.1.1.ipa </string>
       </dict>
       <dict>
         <key>kind</key>
          <string>display-image </string>
          <key>needs-shine </key>
          <true/>
          <key>url </key>
          <string>http://corptravel.qiniudn.com/icon/57x57_fareasthorizon.jpg </string>
        </dict>
        <dict>
          <key>kind </key>
          <string>full-size-image </string>
          <key>needs-shine </key>
          <true/>
          <key>url </key>
          <string>http://corptravel.qiniudn.com/icon/512x512_fareasthorizon.jpg </string>
        </dict>
      </array>
      <key>metadata </key>
      <dict>
        <key>bundle-identifier </key>
        <string>cn.onlinecache </string>

      </dict>
    </dict>
  </array>
</dict>
</plist>

这个文件中，你需要根据的你的需求，替换一些东西，例如：http://corptravel.qiniudn.com/ipa/Corp_FarEastHorizon_3.1.1.ipa，http://corptravel.qiniudn.com/icon/57x57_fareasthorizon.jpg，http://corptravel.qiniudn.com/icon/512x512_fareasthorizon.jpg 以及 dict节点里面的一些信息。

注意哦，这些文件的url都是能访问的，也就是说，是你上传到七牛文件里面之后，所产生的地址（我这里只是测试，上面的url我不能保证任何时候都能访问）

  1. 准备你的html文件，我贴下我的代码：

    <!DOCTYPE html>
    <html>
       <head>
    &lt;meta http-equiv="Content-Type" content="text/html; charset=utf-8" /&gt;
&lt;meta name="viewport" content="width=device-width,minimum-scale=1.0,maximum-scale=1.0,user-scalable=no"/&gt;

       </head>
       <body>
    &lt;p&gt; 
   &lt;a href="itms-services://?action=download-manifest&url=https://dn-corptravel.qbox.me/plist/fareasthorizon.plist"&gt;iOS程序下载 &lt;/a&gt; 
&lt;/p&gt;
&lt;br/&gt;

       </body>
    </html>

    通过这个html文件，我相信你看到了：itms-services://?action=download-manifest&url=https://dn-corptravel.qbox.me/plist/fareasthorizon.plist，我相信苹果开发者们都懂的。如果你不是苹果开发者，我相信你也懂得，所以不解释，反正，这种协议的链接，iOS会认识，它会自动帮你解析。继续看下去，你会发现https://dn-corptravel.qbox.me/plist/fareasthorizon.plist，它就是你上传到七牛的plist的文件地址；但是你发现它是https的，而且，跟你看到地址不一样。这就对了。
    你需要在七牛上设置下：“空间设置” –> “申请域名绑定” –>出现域名域名选择界面–> 选择.qbox.me,因为这个域名支持https协议的；设置完成后，你需要把你手动把https的域名替换掉http的域名，举例如下：
    我的plist的http地址如下：http://corptravel.qiniudn.com/plist/fareasthorizon.plist，替换域名之后的https的地址如下：https://dn-corptravel.qbox.me/plist/fareasthorizon.plist
    我想此时，你会有个疑问：为什么要用https协议呢。原因：因为iOS7.1 要求的

内核-进程-线程他们好抽象

By YYDroid

Published 2012-04-10

我相信很多人“初识”计算机，是因为遇见了Windows的开机画面，是遇见了Word、Execl、Powerpoint（当然老一辈的程序员排除在外，那个时候是“黑屏”或“蓝屏”）


然后，很多人“入门”计算机，是因为下面的一段代码：

#include <stdio.h>
int main(void) 
{    
    printf("Hello World!n"); 
    return 0;
}
在写这段代码的时候，可能那个时候，我们还没有那些复杂的概念——内存、线程等，当然，也还不知道，这段代码是如何转变为机器码的。我还记得我当时的感觉：这段代码，逻辑so easy，会说“Thank you”的人，我想几乎都会懂。当时我的对这段代码的翻译是这样的：include——包含 ；<stdio.h>——啥？一个头文件，哦在头部，所以是头文件；

main——主要的，哦，所以称为main函数；printf——打印，所以，就输出了“Hello World”。我想，这种理解方式，是具体的不能再具体了

再然后，我们学习了操作系统、编译原理、数据结构等一堆课程。此时，我的感受是：“Oh，My god！”其中的概念真多，真抽象。真得让人无法用具体的理解方式去理解计算机。内存是什么？操作系统书籍中的标准概念是：In computing, memory refers to the physical devices used to store programs (sequences of instructions) or data (e.g. program state information) on a temporary . 翻译：在计算机领域，内存指的是，用于临时存储程序和数据的物理设备。如果内存以一种具体的方式理解，该是什么样子的呢？其实，说是内存是物理设备可能还太专业了。咱这辈子没见过物理设备呀。ok！那你就想象它是一个盆，它用来装东西的，当然容量是有限的。

再再然后，在软件开发中、在IT商业应用中，作为一名工程师，我们需要深究这些概念，因为，我们无时无刻都在使用它们。想要用好它们，当然需要深入的了解嘛——知己知彼，百战不殆，貌似夸张了。理解这些概念貌似很困难。为什么这么说呢？以下论据：谁都知道，计算机的世界，是0和1的世界——数学中的0和1。而很多的数学大师们都曾说过，数据研究到最后是哲学（至于哪位大师说过，嘻嘻，我也不知道）。所以，推论是，计算机研究到最后应该也是哲学。读毛泽东的《矛盾论》，读到最后，发现真的很矛盾，因为越读越不懂。我问自己，矛盾这个概念懂了吗，懂了，但是懂与不懂是相互存在的，是个整体，懂存在，所以，不懂依然存在，所以不懂。黑格尔这位大师，又说了，人类从历史中学到的唯一教训，就是人类无法从历史中学到任何教训。


有一天，《深入理解Java 虚拟机》其中一章《Java内存模型与线程》，突然问了自己一个问题：什么是高并发，什么是内核，什么是用户态，什么是内核态？这些概念的理解为什么不是那种感觉呢：当别人问你什么是西瓜时，你直接去买个西瓜，请他吃。可能我没那种感觉，是因为我还没有真正理解它们。但是，我可以确定很难，获取这种感觉很难！为什么呢？因为计算机是个抽象的世界，哲学是个抽象的世界，这个世界本身就是一个抽象的世界。很多人都学会用电，但是什么是电呢？什么是高压电呢？当别人问你这个问题的时候，你总不能让他去碰碰高压电吧（低伏电压当然可以，但是高压电的感觉，我想肯定还是不一样的）。


最后，我得出学习抽象概念的方式：以抽象的思维去学习抽象的知识。我们应该学习梵高，学习那些抽象派的大师，学习那些抽象的思维。不要让自己局限在0和1之间，让自己想象飞起来，你会觉得很美好。


以上言论纯属“自由”范畴，希望你看过后，会“不屑的一笑”！！
